# Cross-Platform Bridging (v3) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Tier 1 shared memory (explicit, facts-only, shared Qdrant collection) + Tier 2 message bridging (instance-id routes, durable outbox, dual transports http/amqp, WhatsApp-safe delivery with summary buffering, attribution with real display names). Version 3.0.0; every feature flag-gated and DEFAULT OFF.

**Architecture:** Spec `docs/superpowers/specs/2026-07-06-bridging-v3-design.md` — its Grounding section carries the exploration's file:line map; its Decisions section is binding (dual transport, instance identity, privacy invariant, held-as-backpressure).

**Tech Stack:** TypeScript (ESM), Node 20+, Zod, Vitest, SQLite (outbox/dedup/buffer tables), Qdrant (shared collection), `amqplib` (NEW dependency, owner-approved 2026-07-06), RabbitMQ 4 (compose `broker` profile), Prometheus metrics.

## Global Constraints

- **Spec is law**, especially: relays deliver as direct messenger sends (never synthetic inbounds); WhatsApp sends ONLY via the platform messenger (auto anti-ban) — `sendControlText` is forbidden in bridge code; `WhatsAppOutboundHeldError` = backpressure → buffer, never blind retry; nothing enters the shared collection except explicitly shared facts; instance ids (not platforms) address everything.
- **All new behavior defaults OFF** (`BRIDGE_ENABLED=false`, `SHARED_MEMORY_ENABLED=false`); the full existing suite must stay green with flags off AND the new suites green with flags on.
- The six v2 volume `name:`s remain byte-identical; new volume `garbanzo-bot-rabbitmq` joins them.
- TS strict, no `any`, ESM `.js`, Pino only, Zod at every boundary (bridge-map file, wire envelopes, env). ~300-line file ceiling.
- **Verify env prefix:** `PATH="$HOME/.local/bin:$PATH" MESSAGING_PLATFORM=whatsapp OWNER_JID=test_owner@s.whatsapp.net OPENROUTER_API_KEY=test_key_ci AI_PROVIDER_ORDER=openrouter`. ALWAYS set it.
- No live broker in CI: amqp tests mock `amqplib`; the transport contract suite runs against both implementations.
- Commits `type(scope): desc`, GitHub noreply author, `npm run check` before each. Never merge — push, PR, owner merges. Reviews: outbound-safety probes MANDATORY on T6/T7/T8; loop/idempotency probes on T5-T8.
